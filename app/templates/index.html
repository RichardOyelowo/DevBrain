Learning signal

The app is built around what learners can inspect after a quiz.

A quiz should not end with only a score. DevBrain treats every attempt as a saved learning record that can be reviewed, filtered, and used to pick the next session.

Attempt Topic, difficulty, count, score, grade, and completion time are saved.
Answer Each selected answer is stored against the correct answer and explanation.
Dashboard Progress stats summarize total quizzes, average score, strongest topic, and weakest topic.
History Completed attempts can be filtered and reopened for review instead of disappearing.

Content model

Categories and languages are separate, so practice stays flexible.

A category describes the skill being tested. A language describes the syntax used in a code snippet. That means Data Structures can include Python today, JavaScript tomorrow, and another language later without rebuilding the quiz flow.

Built for the MVP stage

The product keeps the architecture simple while still feeling like a real app.

DevBrain uses Flask, Jinja, SQLAlchemy, migrations, and a small amount of JavaScript. That keeps the MVP easy to reason about while still supporting accounts, dashboards, admin tools, imports, and saved quiz attempts.

Server-rendered

No frontend build step

Templates and CSS carry the product experience without adding a second frontend app too early.

Database-owned

No external quiz dependency

Published questions come from DevBrain's own database, so content quality stays under admin control.

Admin-ready

Content can grow from the UI

Categories, languages, questions, imports, and fast starts are editable without manual database work.

Question bank

Owned content instead of a random external feed.

Every question is stored in the database with a category, difficulty, explanation, answer options, optional language, and optional code snippet. Admins can import batches as drafts, review them, and publish only the questions that fit the product.

Draft

Import without publishing

Batch imports enter the library as drafts, which keeps quality review in the admin workflow.

Pick

Randomized from the database

Quiz questions are selected from published matches and shuffled so sessions do not simply follow entry order.

Grow

Designed for deeper tracks

The same model can support broad review now and language-specific coding tracks later.

Quality loop

Questions are drafted, reviewed, published, and measured.

The import workflow is intentionally conservative. Imported questions become drafts first, so admins can fix wording, difficulty, answers, explanations, language tags, and snippets before users see them.

1

Import or create

Add questions manually or paste a structured JSON batch into the admin importer.

2

Review as draft

Drafts stay out of quizzes until the admin checks answer quality and explanations.

3

Publish to the bank

4

Improve from usage

As attempts build up, weak areas and thin categories become easier to spot.
